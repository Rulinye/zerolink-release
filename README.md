# zerolink

本仓库托管 zerolink 客户端的安装包。**当前版本：0.9.125**

## Note.
链路名称格式为“入口-出口”，如 "gz-cc" 即入口在 gz，出口在 cc，除非使用 direct 后缀的链路（代表直连，一般都是无视地理的优化专线）。大陆用户应选择大陆入口，海外用户应选择海外入口。
- gz = 广州
- cc = 韩国春川
- hk = 香港
- seoul = 韩国首尔
- jp = 日本
- sg = 新加坡
- usa = 美国西部
- **后缀为“direct”的链路为直连，任何地理位置的用户都可选择**

**严禁！！！无端跨地域选择链路，检测封号将不提醒。**

## Windows 安装 / 卸载

### 安装

双击下载的 `zerolink-installer_0.9.125_x64.exe` → UAC 弹出点“是” → 安装完成。系统服务在安装时自动注册，无需额外操作。

> SmartScreen 提示“未知发布者”：点“更多信息” → “仍要运行”。我们暂未购买 EV 签名证书。
>
> 安装时可能有一两个黑色 cmd 窗口一闪而过，这是 Windows 调用命令行工具的标准行为，不影响功能。
>
> 如果启动应用后主页提示“系统组件未安装”（自动注册被杀软拦截等罕见情况），点界面里的“立即安装”按钮 → UAC 确认即可。

### 卸载

**方式 1. 在安装的文件夹目录直接运行 `uninstall.exe`。** 可能会出现黑框，耐心等待即可。

**方式 2. Settings → 应用 → zerolink → 卸载**（或控制面板“程序和功能”）。会自动：

- 停止并删除 Helper 服务
- 卸载我们安装的 TAP-Windows 网卡驱动（不动其他 VPN 软件的）
- 删除入站防火墙规则
- 清理安装目录 + 用户数据目录

## macOS 安装 / 卸载

### 安装

1. 下载 `zerolink.app.tar.gz`，在访达里双击解压得到 `zerolink.app`，拖到“应用程序”。
2. 首次启动时会弹 macOS 授权框要求 sudo 密码，输入即可。系统组件会装到 `/Library/PrivilegedHelperTools/` + `/Library/LaunchDaemons/`。

> Gatekeeper 提示“无法验证开发者”：系统设置 → 隐私与安全性 → 找到 zerolink 条目 → 点“仍要打开”。我们暂未做 Apple notarize。
>
> 目前仅支持 Apple Silicon（M 系列芯片）。

### 卸载

应用内：**设置 → 完全卸载系统组件**（红色按钮），输入 sudo 密码，然后把 `应用程序/zerolink.app` 拖到废纸篓即可。

## 规则分流

主页“规则”标签页管理分流配置。每个**配置文件**包含一组规则，可随时切换激活的配置文件。

### 默认配置

应用内置 `GeoIP-CN` + `GeoSite-CN` 规则集：**中国大陆 IP / 网站直连，境外走代理**。这是大多数用户最实用的策略，首次启动后无需手动调整即可使用。

### 三种模式

| 模式 | 含义 | 适用场景 |
|---|---|---|
| **规则 (黑名单)** | 默认走代理，命中规则的走直连 | 全局代理，但少数应用需要直连(银行、本地服务) |
| **规则 (白名单)** | 默认直连，命中规则的走代理 | 大部分流量直连，只为特定网站 / 应用走代理(节省带宽 / 降低代理负载) |
| **全局** | 所有流量走代理，忽略规则 | 临时全代理，不需要分流逻辑 |

### 添加自定义规则

每条规则有三个字段：

- **类型**: 域名 / 域名后缀 / 域名关键词 / IP / IP 段 等
- **值**: 比如 `example.com`、`192.168.1.0/24`
- **动作**: 代理 / 直连 / 阻断

例如想让 `*.github.com` 强制走代理，加一条 “域名后缀 = `github.com` → 动作 = 代理”。

## 联机房间

应用提供基于服务器中转的虚拟局域网，房友之间像在同一台路由器下一样互相访问。常见用途: LAN 联机游戏、文件共享、内网穿透、远程桌面发现。

### 房间类型

| 类型 | 说明 | 平台 |
|---|---|---|
| **L3 (默认)** | 三层路由，基于 IP 转发，广播 / 多播由服务器转发 | Windows + macOS |
| **L2** | 真二层桥接，以太网帧 / ARP / 原生广播。游戏的 LAN 自动发现、Wake-on-LAN 等需要 L2 的协议都正常 | **仅 Windows**(macOS 因平台限制不支持) |

绝大多数场景 L3 已经够用。只有玩需要原生广播的局域网游戏(部分老游戏的“局域网房间”列表)才需要 L2。

### 联机模式

| 模式 | 说明 | 适用 |
|---|---|---|
| **中转 (默认)** | 数据走联机节点中继，抗 NAT / DPI，稳定 | 大多数情况 |
| **直连** | 尝试 P2P 直连，延迟更低 | 双方 NAT 类型友好，追求最低延迟 |

### Broker 路径

控制信道(房间状态、成员变更等小数据)的路径，只影响**你自己**：

- **直连 broker (默认)**: 控制面直接到联机节点，延迟最低
- **中转 broker**: 控制面绕一跳到指定节点，适合直连 broker 受阻的网络环境
